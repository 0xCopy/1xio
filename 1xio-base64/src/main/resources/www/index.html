Base-64 and Hex codecs in the form of Character encodings.

The inspiration for this comes from the RFE 4235519 which request proper access to Base-64 encoding and decoding as part of the java libraries. People have been using a codec in sun.misc.*, but this was never supported and, lo and behold, the API has disappeared in Tiger.

These codecs translate between byte sequences and characters so, in many ways, the natural place for them is in the character encoding system. In a real sense they're just another interpretation of binary data as characters.

By creating an approprate CharsetProvider class which can be referenced from the META-INF/services directory this library enables the codecs to be invoked just by using their names, especially in String.getBytes and the String constructor from bytes.

The most important non-standard behaviour of these codecs is that, in the event that the characters supplied to the Encoder not being valid code they throw a IllegalArgumentException, rather that substitute a "?" byte, a useless behaviour in this context. I'd rather have thrown an appropriate exception from the java.nio.charset set, but unfortunately the conversion methods in String intercept these.

The CharsetProvider treat the encoding "name" passed in a rather special way, gleaning extra information from it.

base64
Generates standard base64 without whitespace.
base64:nn
Where nn is a number between 4 and 1024, inserts a line break every nn characters when encoding.
hex
Generates standard hex using lower case letters a-f.
HEX
Generates hex using upper case letters for A-F.
hex:nn
Generates hex with line breaks every nn characters.

When compiling these, remember the services directory must be copied to the META-INF in the build/classes or whatever, where it will be incorporated in the jar.
